Case Study



Meetup



Project Overview



What is it?

Meetup is a serverless, progressive web application (PWA) built with React using a test-driven development (TDD) technique. The application uses the Meetup API to fetch upcoming events for a selected city.

What was the objective?

To create a responsive serverless PWA with offline support, which included data visualization in the form of charts and a OAuth2 authentication flow, as part of the CareerFoundry Web Development course.


Which tools were used?


Client-side: React, Bootstrap, Recharts library
Development Environment: Node.js
Hosting: Github pages






Building The App



The technical requirements for creating the front-end:

  • The app ​must b​e a React application.
  • The app ​must b​e built using the TDD technique.
  • The app ​must​ use the Meetup API and OAuth2 authentication flow.
  • The app ​must u​se serverless functions (AWS lambda is preferred) for the authorization server instead of using a traditional server.
  • The app’s code ​must b​e hosted in a Git repository on GitHub.
  • The app ​must w​ork on the latest versions of Chrome, Firefox, Safari, Edge, and Opera, as well as on IE11.
  • The app ​must d​isplay well on all screen sizes (including mobile and tablet) widths of 1920px and 320px.
  • The app ​must​ pass ​Lighthouse​’s PWA checklist.
  • The app ​must​ work offline or in slow network conditions with the help of a service worker.
  • Users ​may​ be able to install the app on desktop and add the app to their home screen on mobile.
  • The app ​must​ be deployed on GitHub Pages.
  • The API call ​must​ use React axios and async/await.
  • The app ​must​ implement an alert system using an Object-Oriented Programming approach to show information to the user.
  • The app ​must​ make use of data visualization (recharts preferred).
  • The app ​must​ be covered by tests with a coverage rate >= 90%.
  • The app ​must​ be monitored using an online monitoring tool.

MongoDB.


The database for the API was built using MongoDB, which is a NoSQL non-relational database program that stores data using collections, documents, and schemas, and tends to be more flexible than the traditional, SQL-based, relational database structures.



Testing with Postman.


The API endpoints were tested on Postman, an interactive tool that I used for verifying the HTTP methods: GET, POST, PUT, and DELETE. In the screenshot, the result for the POST request "add user" can be seen. In this test result, it can also be seen that the password has been correctly hashed adhering to the application's security regulations logic.




Client-Side:


The technical requirements for creating the front-end:

  • The application must be a single-page application (SPA).
  • The application must use state routing to navigate between views and share URLs.
  • The application must give users the option to filter movies.
  • The application must give users the option to sort movies.
  • The application must initially use Parcel as its build tool.
  • The application must be migrated to create-react-app.
  • The application must be written using the React library and in ES2015+.
  • The application must be written with React Redux.
  • The application must use Bootstrap as a UI library for styling and responsiveness.
  • The application must contain a mix of class components and function components.
  • The application may be hosted online.

Challenges Faced


This was the most complete project from my Full-Stack Web Development course at CareerFoundry. The objective was to build a complete full-stack application from scratch. It was quite daunting at the start of the project given that there was a mountain of new technologies ahead, and I knew that the learning curve was going to be steep. Nevertheless, after many head-scratching hours sat in front of the screen, and thanks to the course content, the hands-on experience, on-line investigation, self-learning, and dedication, I was able to scale that mountain, and competently incorporate all the server-side and client-side technical requirements that are listed in the "Building The App" section above.

All in all, the learning experience was very positive, and I was extremely happy with the final result. I achieved exactly what I had set out to do: build an error-free application with beautiful UI, become more competent with React, consolidate my full-stack skills, and add more tools to my programming tool-belt!
