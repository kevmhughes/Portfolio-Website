A Case Study



MyFlickz



Project Overview



What is it?

MyFlickz is a responsive web application built with the MERN stack. It allows the user to create a user profile, and access a database of movies. The user can filter a movie by name, and choose to look at more detailed information about a selected movie, director, or genre. The user is also able to update their user profile data, create a list of their favourite movies, and delete their profile.

What was the objective?

To create a RESTful full-stack application from scratch using JavaScript technologies as part of the CareerFoundry Web Development course.


Which tools were used?

Stack: MERN (MongoDB, Express, React, Node.js)
Server-side: MongoDB, Mongoose, Express
Client-side: React, Redux, Bootstrap
Development Environment: Node.js
Hosting: Heroku






Building The App



Server-Side:


The technical requirements for creating the back-end:

  • The API must be a Node.js and Express application.
  • The API must use REST architecture.
  • The API must use at least three middleware modules, such as the body­parser package for reading data from requests, and morgan for logging.
  • The API must use a “package.json” file.
  • The database must be built using MongoDB.
  • The business logic must be modeled with Mongoose.
  • The API must provide movie information in JSON format.
  • The API must be tested in Postman.
  • The API must include user authentication and authorization code.
  • The API must include data validation logic.
  • The API must meet data security regulations.
  • The API source code must be deployed to a publicly accessible platform like GitHub.
  • The API must be deployed to Heroku.


GitHub.


The source code is deployed to GitHub, a platform for hosting, collaborating, and version control. Take a look at the code here.



MongoDB.


The database for the API was built using MongoDB, which is a NoSQL non-relational database program that stores data using collections, documents, and schemas, and tends to be more flexible than the traditional, SQL-based, relational database structures.



Testing with Postman.


The API endpoints were tested on Postman, an interactive tool that I used for verifying the HTTP methods: GET, POST, PUT, and DELETE. In the screenshot, the result for the POST request "add user" can be seen. It can also be seen in this result that the password has been correctly hashed adhering to the application's security regulations logic.




Client-Side:


The technical requirements for creating the front-end:

  • The application must be a single-page application (SPA).
  • The application must use state routing to navigate between views and share URLs.
  • The application must give users the option to filter movies.
  • The application must give users the option to sort movies.
  • The application must initially use Parcel as its build tool.
  • The application must be migrated to create-react-app.
  • The application must be written using the React library and in ES2015+.
  • The application must be written with React Redux.
  • The application must use Bootstrap as a UI library for styling and responsiveness.
  • The application must contain a mix of class components and function components.
  • The application may be hosted online.

Challenges Faced


This was the most complete project from my Full-Stack Web Development course at CareerFoundry. The objective was to build a complete full-stack application from scratch. It was quite daunting at the start of the project given that there was a mountain of new technologies ahead, and I knew that the learning curve was going to be steep. Nevertheless, after many head-scratching hours sat in front of the screen, and thanks to the course content, the hands-on experience, on-line investigation, self-learning, and dedication, I was able to scale that mountain, and competently incorporate all the server-side and client-side technical requirements that are listed in the "Building The App" section above.

All in all, the learning experience was very positive, and I was extremely happy with the final result. I acheived exactly what I set out to do: build an error-free application with beautiful UI, become more competent with React, consolidate my full-stack skills, and add more tools to my programming tool-belt!
